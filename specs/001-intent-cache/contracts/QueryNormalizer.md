# 契约：QueryNormalizer

**对齐**: AI_ARCHITECTURE.md v3.5
**包**: `com.mrshudson.optim.intent.cache`

---

## 概述

`QueryNormalizer` 按照 AI_ARCHITECTURE.md 的规定，对用户输入进行规范化以实现一致的意图指纹生成。它将相对时间表达式转换为具体日期，以实现精确的缓存匹配。

---

## 接口

```java
public interface QueryNormalizer {

    /**
     * 规范化用户输入以实现一致的指纹生成
     * @param input 原始用户输入
     * @return 带具体日期的规范化字符串
     */
    String normalize(String input);

    /**
     * 从规范化输入生成指纹哈希
     * @param normalizedInput 规范化文本
     * @return MD5 哈希字符串
     */
    String generateFingerprint(String normalizedInput);

    /**
     * 获取用于时间规范化的具体日期
     * @return 使用的日期 (规范化时的当天日期)
     */
    LocalDate getConcreteDate();
}
```

---

## 规范化规则

### 时间规范化 (AI_ARCHITECTURE.md 要求)

根据 AI_ARCHITECTURE.md v3.5 规范，将相对时间转换为**具体日期**，便于精确匹配：

| 原始表达 | 转换示例 (假设今天是 2026-04-04) |
|---------|-------------------------------|
| 今天 | 2026-04-04 |
| 昨天 | 2026-04-03 |
| 明天 | 2026-04-05 |
| 后天 | 2026-04-06 |
| 刚才/刚刚 | 2026-04-04 (带时间精度标记) |
| 现在/目前 | 2026-04-04 |
| 这周/本周 | 2026-04-04 (周起始日) |
| 下周 | 2026-04-11 (下周起始日) |

> **注意**: AI_ARCHITECTURE.md v3.5 明确采用**具体日期方案**而非占位符方案

### 口语规范化 (可选)

将口语表达映射为标准形式：
- 查一下, 看看 → 查询
- 帮我弄 → 创建
- 给我 → 提供
- 能给我 → 请提供

---

## 示例

```
输入: "今天北京天气怎么样"
具体日期: 2026-04-04
规范化后: "2026-04-04北京天气怎么样"
指纹: "md5(2026-04-04北京天气怎么样)" → "a1b2c3d4..."
```

---

## 设计原理

- 采用**具体日期方案**而非占位符 ([TIME])，与 AI_ARCHITECTURE.md v3.5 保持一致
- 相对时间词汇在每次查询时解析为当前日期
- 缓存条目按自然日隔离，避免跨天数据污染
- 同一天内相同查询能精确命中缓存
- 跨天查询自动使用新的日期，保证语义正确性

---

## 与 AI_ARCHITECTURE.md 对齐

| 架构规范 | 实现 |
|---------|------|
| "今天天气怎么样" → "2026-03-23天气怎么样" | ✓ 具体日期转换 |
| 将相对时间转换为具体日期，便于精确匹配 | ✓ 精确匹配策略 |
| QueryNormalizer 位于 IntentCacheStore 之前 | ✓ 调用链顺序 |
